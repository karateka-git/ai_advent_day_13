# Documentation Policy

В проекте `ai_advent_day_11` для нового кода нужно по умолчанию добавлять KDoc.

Правило для Codex:
- документировать все новые публичные сущности и новые публичные функции;
- документировать новые нетривиальные внутренние функции, если они управляют памятью, prompt assembly, storage, mapping, lifecycle или другой orchestration-логикой;
- для функций с параметрами и возвращаемым значением по умолчанию добавлять `@param` и `@return`, если это делает контракт быстрее читаемым;
- не оставлять новый важный код без документации с расчётом “дописать потом”.
