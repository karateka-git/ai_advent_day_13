# Agent Preflight

Короткий operational summary для работы в этом проекте.

Использование:

- в новой сессии после ознакомления с локальным и общим `MemoryBank`;
- перед новым смысловым блоком, если дальше ожидаются новые правки в коде или файлах.

## Обязательная проверка перед новыми правками

1. Ознакомиться с локальным `MemoryBank` проекта.
2. Ознакомиться с общим `MemoryBank`.
3. Свериться с этим preflight-документом.
4. Проверить `git status`, если впереди новый смысловой блок с правками.
5. Если есть незакоммиченные изменения и начинается новый смысловой блок с новыми правками, напомнить пользователю про коммит и дождаться явного решения.

## Критичные правила проекта

- По умолчанию не добавлять backward compatibility со старыми форматами и legacy-поведением без явной просьбы пользователя.
- Для нового публичного кода и нового важного orchestration-кода по умолчанию добавлять KDoc.
- Не трогать русские строки без явной необходимости; при правках рядом с кириллицей следить за корректной UTF-8 кодировкой.
- Если задача связана с пользовательскими строками, debug-выводом или prompt preview: сначала посмотреть текущее содержимое файла, править маленькими точечными изменениями, затем отдельно синхронизировать тесты.
- Для новых пользовательских, debug- и prompt-строк по умолчанию использовать русский язык, если пользователь не просил иначе.

## Коммит-правило

- Если пользователь переключается на новый смысловой блок и впереди новые правки, сначала проверить, есть ли незакоммиченные изменения.
- Если изменения есть, мягко предложить сделать коммит перед продолжением.
- Не навязывать коммит после каждой мелкой правки.
- Если пользователь хочет только анализ, объяснение, проверку, запуск команд, сборку, просмотр логов или ручной сценарий без новых правок, отдельно не останавливать работу из-за коммита.

## PowerShell workflow для этого проекта

- Команда `собери проект` означает полный workflow: очистка runtime-истории в `config/conversations/` -> `.\gradlew.bat build` -> `.\gradlew.bat installDist` -> запуск проекта в новом окне PowerShell.
- Команда `запусти проект` по умолчанию означает только запуск уже собранной версии, если нет признаков несвежей сборки.
- Команда `собери сравнение` означает: очистка comparison-артефактов -> `.\gradlew.bat test` -> `.\gradlew.bat compareStrategies`.
- Команда `запусти сравнение` по умолчанию означает только запуск `compareStrategies` в новом окне PowerShell.
